# Review Checklist

Use this before the final response.

- Existing patterns inspected.
- Design profile formed.
- Existing components reused where practical.
- Existing tokens, classes, variants, or utilities reused where practical.
- New styles are necessary, minimal, and justified by local conventions.
- Visual rhythm, density, spacing, and alignment are preserved.
- Typography hierarchy is preserved.
- Color usage matches semantic roles and brand conventions.
- Hover, focus, active, selected, disabled, loading, empty, error, and success states are handled when relevant.
- Responsive behavior is preserved across likely viewport sizes.
- Accessibility semantics, labels, keyboard behavior, focus visibility, reduced motion, and contrast are preserved.
- No new dependencies were introduced unless necessary and approved.
- No parallel design system or duplicated component was created.
- Relevant tests, linting, type checks, story/visual checks, or build commands were run when appropriate and allowed.
- Any checks not run are clearly stated.
- Remaining risks or uncertainty are disclosed.

If visual verification was not performed, say so plainly.
